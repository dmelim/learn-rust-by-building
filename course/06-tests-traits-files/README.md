# Module 6 — Tests, Traits, and Files

## Outcome

Define reusable record behavior, test public APIs, and persist the delivery
queue to disk.

This module uses [Rust Book Chapter 10](https://doc.rust-lang.org/book/ch10-00-generics.html),
[Chapter 11](https://doc.rust-lang.org/book/ch11-00-testing.html), and standard
library file-I/O documentation.

## Lesson sequence

| Lesson | Main idea | Checkpoint |
|---|---|---|
| [1. Generics](01-generics.md) | Reusable type parameters | Generic storage functions |
| [2. Traits](02-traits.md) | Shared behavior contracts | Define `Record` |
| [3. Lifetimes](03-lifetimes.md) | Relationships between borrows | Return customer views |
| [4. Testing](04-testing.md) | Unit and integration boundaries | Test public behavior |
| [5. File I/O](05-file-io.md) | Read, write, and contextual errors | Persist records |
| [Practical](06-practical-persistence.md) | Combined persistence | Save and load a queue |
